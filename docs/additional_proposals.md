## 1. 目的

現時点で出ている追加案を一覧化し、優先度と着手順を整理する。

---

## 2. 追加案一覧

### A. Release 同期機能（最優先）

ステータス: 実装済み（v1）

- 内容:
  - `tool_versions` / `tool_assets` を GitHub Releases から自動同期する
  - 手動登録運用を減らす
- 最小要件:
  - 管理者が「今すぐ同期」を実行できる
  - 同期結果（追加件数・失敗理由）を確認できる
- 期待効果:
  - 配布データ更新の手作業を削減

### B. 複数 Organization / Repository 対応

ステータス: 実装済み（v1: 1ツール=1リポジトリ前提）

- 内容:
  - 対象 org/repo を増やしても同じ仕組みで運用できるようにする
- 検討項目:
  - 単一 GitHub App で統一するか
  - `tool_repositories` に同期状態管理項目を追加するか（`last_synced_at` など）
- 期待効果:
  - 将来の拡張時に再設計コストを下げる

### C. 管理画面（最小）

ステータス: 実装済み（v1）

- 内容:
  - 同期実行ボタン
  - 同期ステータス表示
  - ツール有効/無効の切り替え
- 期待効果:
  - 運用担当者がCLIなしで管理可能

### D. ダウンロード監視強化

- 内容:
  - `download_logs` の一覧表示
  - 失敗理由の可視化
  - 失敗率の把握
- 期待効果:
  - 障害切り分けを高速化

### E. GitHub Actions 運用整備

- 内容:
  - CI（build/test）を PR で実行
  - Supabase 反映フロー（dev自動、prod手動承認）を整備
- 期待効果:
  - 手動反映ミスを減らす
